高级类型

交叉类型

交叉类型将多个类型合并为一个类型，包含了所有类型的特性

函数似乎会被后面的定义覆盖，而属性不会，具体行为在实践中再去探索吧

联合类型

A|B[..]
只能访问联合类型成员的交集
因此访问某个类型的特有方法或者属性时，可能会报错，我们就需要多次使用断言，这显然会产生代码冗余，不过ts已经考虑到这一点，为我们提供了类型保护机制

下面使用用户自定义保护机制来使代码逻辑更清晰

这样调用就没问题了，下面来看看TS自带的类型保护

typeof类型保护

typeof类型保护只有两种写法，typeof value ===|!== typename,而且只能判定原始类型

instance类型保护

instanceof的右侧要求是一个构造函数，TypeScript将细化为：

此构造函数的 prototype属性的类型，如果它的类型不为 any的话
构造签名所返回的类型的联合

null 与 undefined

任何类型都可以赋值为null或者undefined，ts是如此设计的。但是可以在配置时，或者运行命令时，添加--strictNullChecks来避免这样的错误，当你想对某个类型赋值null时，必须要显示的声明它，如：let sn:number|null。

当使用这个附加选项时，可选参数和可选属性会自动联合undefined类型

类型保护和类型断言

当你想判断或者去除null类型时，与js过往行为相同，只需判定是否相等即可，无需像其他类型一样自定义保护类型或者typeof&instanceof

类型别名

类型别名时一种引用，语法：type 类型 = Name 仅此而已
